抽象工厂模式介绍
    抽象工厂模式一般用于具有产品树和产品族的场景下。
   1、抽象工厂模式：定义了一个interface用于创建相关或有依赖关系的对象簇，而无需指明具体的类。
   2、抽象工厂模式可以将简单工厂模式和工厂方法模式进行整合
   3、从设计层面来看，抽象工厂模式就是对简单工厂模式的改进（或者进一步的抽象）。
   4、工厂抽象成两层，AbsFactory（抽象工厂）和具体实现的工厂子类。
   程序员可以根据创建对象类型使用对应的工厂子类。这样讲单个的工厂类变成了工厂簇，更利于代码的维护。

缺点：
    增加代码复杂度：
    如果新加产品树，需要增加三个产品类，譬如 ：【卡车，奥迪卡车，奔驰卡车】
    并且要修改三个工厂，【汽车制造工厂，奥迪制造工厂，奔驰制造工厂】
